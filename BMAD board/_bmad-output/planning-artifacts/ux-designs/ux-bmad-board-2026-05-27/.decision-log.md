## D2 — Visual Style Direction
- **Date:** 2026-05-27

## D3 — Dual Theme Requirement
- **Date:** 2026-05-27
- **Decision:** Both dark and light themes required. Dark is primary/showcase; light must be fully supported.
- **Context:** User explicitly requested dark theme with turquoise accents AND a light theme.

## D4 — Current Pain Point: Primitive Design
- **Date:** 2026-05-27
- **Decision:** Current UI uses a flat, Jira-mimicking palette (`jira-gray-*`, `jira-blue`) with minimal depth, rounded-[8px]-only badges, no dark mode, hardcoded colors in globals.css vars. All components use inline Tailwind utility classes with `jira-*` custom colors. No design system — 4 ad-hoc components (Sidebar, StatusBadge, CreateModal, Providers).
- **Context:** Code audit confirmed: `globals.css` defines only 5 CSS vars on `:root`, all light-mode. `tailwind.config.ts` extends colors with `jira.*` palette (flat corporate Jira colors). No theme switching mechanism exists.

## D5 — Code Block Highlighting
- **Date:** 2026-05-27
- **Decision:** Replace current `prose-code:text-pink-600` (raw pink inline code) and `prose-pre:bg-jira-gray-900` (monochrome blocks) with Catppuccin-based syntax highlighting. Light: Catppuccin Latte palette for inline/blocks. Dark: Catppuccin Mocha. Inline code: teal accent (#0F766E light, #2DD4BF dark) on tinted background. Implementation: highlight.js or Shiki.
- **Context:** User reported code highlighting in parsed documents is "just red text" and looks bad. Code audit confirmed: `prose-code:text-pink-600` and `prose-pre:bg-jira-gray-900 prose-pre:text-gray-100` with no syntax highlighting.

## D5 — Mermaid Diagram Rendering
- **Date:** 2026-05-27
- **Decision:** All ````mermaid` fenced code blocks in markdown documents and stories must render as SVG diagrams via Mermaid.js (client-side). Visual treatment: theme-aware colors (foreground-primary for labels, accent for highlights, border-default for edges/shapes), `code-block-bg` container background, `rounded.lg`, language badge "mermaid" top-left. Light theme: white SVG canvas. Dark theme: `surface-elevated-dark` (#181B23) SVG canvas. Failure path: raw monospace code block + `destructive` inline error banner.
- **Context:** User reported Mermaid diagrams are common in BMAD markdown artifacts. Current implementation renders them as raw text code blocks with no diagram visualization.

## D6 — Icon System: Lucide replaces heroicons + emoji
- **Date:** 2026-05-27
- **Decision:** Replace `@heroicons/react` and all emoji-based icons (⚡📖✅🎯📋📄😿) with Lucide icon system (stroke-based, outline style, 1.5px stroke width). 18px default, 16px inline, 22-24px stat cards. Empty states use Lucide icons inside `rounded.lg` tinted container — never bare emoji. This is a hard visual identity rule: zero emoji-as-icons.
- **Context:** Current codebase uses `@heroicons/react` SVGs in sidebar and inline emoji decorators in stat cards and empty states. User described them as "убогие" (shabby/inferior). Lucide provides a modern, consistent stroke-based icon set that aligns with the Win11/macOS aesthetic.
