# Product Brief: BMAD Board

## Executive Summary

BMAD Board — десктопное Electron-приложение для парсинга, визуализации и редактирования BMAD-артефактов (Markdown-файлов). Существующее веб-приложение (Next.js + React) уже парсит MD-файлы, отображает статусы на agile board и позволяет редактировать артефакты. Следующий шаг — миграция в Electron для поддержки нескольких проектов с хранением настроек в SQLite, автосинхронизации с файловой системой, тёмной темы в стиле Windows 11 и полного цикла редактирования. Продукт распространяется под лицензией MIT, монетизация через донаты (GitHub Sponsors, Buy Me a Coffee).

## Vision

BMAD Board станет стандартным инструментом для команд, работающих с BMAD-методологией и AI-агентами. В перспективе 2–3 лет:
- Плагины и расширения для кастомизации
- Интеграция с популярными AI-агентными фреймворками
- Активное open source сообщество контрибьюторов
- Устойчивая модель донатов для поддержки разработки

## The Problem

Веб-приложение неудобно для повседневной работы с BMAD-артефактами. Пользователям нужно:
- Открывать браузер и запускать dev server
- Вручную синхронизировать изменения — агенты меняют файлы, но приложение не видит обновлений
- Работать только с одним проектом
- Пользоваться веб-интерфейсом без тёмной темы и нативного UX
- Проверять работоспособность edit-функций, которые могут не работать корректно

Для команд, использующих BMAD-методологию, это создаёт трение между AI-агентной работой и визуальным контролем артефактов.

## The Solution

Десктопное Electron-приложение, которое:
- Переиспользует существующий React-код (Next.js → Electron + React)
- Автоматически отслеживает изменения в файловой системе (интервал ~30 сек)
- Хранит настройки проектов в SQLite (пути к артефактам, предпочтения)
- Предоставляет тёмную тему с дизайном в стиле Windows 11
- Гарантирует работоспособность всех edit-функций с предупреждением о ручной правке
- Бесплатно обновляется через electron-builder и GitHub Releases

## Key Differentiators

- **AI-first редактирование** — AI-агентное редактирование как основная практика, ручное редактирование только в крайних случаях с явным предупреждением
- **Файловая система как источник правды** — никаких проприетарных форматов, только Markdown
- **Open source + донаты** — лицензия MIT, бесплатное использование, монетизация через сообщество
- **Нативный десктопный UX** — полноценное десктопное приложение, а не обёртка сайта

## Target Audience

**Primary:** Разработчики и команды, использующие BMAD-методологию, которые работают с AI-агентами и нуждаются в визуальном контроле и лёгком редактировании артефактов.

**Secondary:** Open source сообщество, которое может форкать, контрибьютить и использовать приложение для своих проектов.
